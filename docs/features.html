Features

LAME as quality mp3 encoder

The LAME encoder has a good reputation in the audio scene to be the best MP3 encoder currently available. It supports gapless encoding, generates VBR Info tag with infos about the encoded audio track, and automatically applies Track ReplayGain.

Modern audio encoder

winLAME supports the current state-of-the-art audio formats, most of them both for encoding and decoding. Among them are:

  • LAME MP3 encoder
  • Opus Codec
  • Ogg Vorbis
  • FLAC
  • Speex
  • Monkey's Audio

Many other audio formats

winLAME supports many other audio formats, both for reading in and also for encoding. winLAME uses the mature libSndfile library from Erik de Castro Lopo and the BASS audio library to support these audio formats. Supported file extensions are: WAV, AIFF, AU, AVR, CAF, FLAC, HTK, IFF, MAT, MPC, OGA, PAF, PVF, RAW, RF64, SD2, SDS, SF, VOC, W64, WVE, XI, MOD, S3M, XM, IT, MTM, MO3, UMX, WMA, ASF (phew!)

Multicore CPU support

winLAME uses your multicore CPU to encode multiple audio tracks in parallel, leading to faster encode times. You can enqueue the next batch of audio tracks while winLAME is still working on encoding the previous audio tracks. The task scheduler starts new encoding tasks when the CPU is ready again.

CD Extraction

winLAME supports reading audio tracks from CDs. The freedb.org database and Cover Art Archive can be used to get track and artist information about the CDs, so that you don't have to type in the track names. The format for track names can be customized to the filename schema that you prefer.

Encoder presets

winLAME provides a set of preset settings for each encoder so that you don't get lost in the many settings that each encoder provides. This way you get quality encoded files. Most presets come from recommendations from the audio encoding scene, e.g. Hydrogenaudio.

Playlist support

winLAME supports reading and writing playlist files, for specifying input files, as well as creating an output playlist for your encoded tracks. The supported input formats are: PLS, M3U and CUE. The output playlist is always an M3U file.

ID3 Tag support

winLAME can transfer tag metadata from audio tracks (ID3 tags, ID3v2 tags, WAVE INFO chunk, APE tags) to the encoded file, preserving the metadata from the original files. The following infos are transferred: Track title, Artist, Disc artist, Album name, Track comment, Genre, Year of recording and Track number.

Many other features

winLAME features many other functions, e.g.:

  • Explorer context menu
  • Portable version
  • Recursive file system encoding
  • German translation
To learn more about the functions in winLAME, press F1 when winLAME is running to open the Help file that contains the manual.
